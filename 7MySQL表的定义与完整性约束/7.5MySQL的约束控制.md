# [[2.2数据模型#3 数据的完整性约束|数据库的完整性约束]]
## (1)实体完整性
强制表中标识符列或主键的完整性
# MySQL中常用约束

## 1.主键约束
用于定义基本表的主键，起唯一标识作用(不能为NULL，不能重复)

用于列约束的定义
```sql
字段名
```

单个主键
[例7-18]建立一个Stu表，定义Sno为Stu的主键。
```sql
CREATE TABLE Stu
Sno char(6) PRIMARY KEY,
Sname char(8),
Ssex char(2),
Zno varchar(20));
```
or
```sql
CREATE TABLE Stu
Sno char(6) NOT NULL,
Sname char(8),
Ssex char(2),
Zno varchar(20),
PRIMARY KEY(Sno)
);
```

多个主键
[例7-20]建立一个SC表，主键由Sno和Cno共同构成。
```sql
create table sc
sno char(12),
cno char(10) , 
score int,
primary key(sno,cno)
);
```
